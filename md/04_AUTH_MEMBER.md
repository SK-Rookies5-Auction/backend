# 👤 04. 회원 인증 및 마이페이지 상세 명세 (Auth & Member)

본 문서는 사용자의 보안 인증, 권한 관리 및 마이페이지 내 활동 내역 조회를 위한 시스템 로직을 정의합니다. AI는 이 명세를 바탕으로 SecurityConfig, JWT Provider, 마이페이지 Controller를 설계해야 합니다.

---

## 1. 회원 인증 및 인가 보안 정책 (Authentication & Authorization)

### 1.1 기술 스택 및 아키텍처
- **Framework**: Spring Security 6.x (Spring Boot 3.x)
- **Authentication**: JWT (JSON Web Token) 기반 무상태(Stateless) 인증
- **Encryption**: 비밀번호 저장 시 BCrypt 해시 알고리즘을 통한 단방향 암호화 필수
- **Token 관리**: Access Token을 통한 API 요청 인가 처리

### 1.2 보안 로직 상세
- **API 권한 분리**: 유저 역할(ROLE_USER, ROLE_ADMIN)에 따른 엄격한 접근 제어 적용
- **토큰 검증**: 모든 API 요청은 인증 필터(JwtAuthenticationFilter)를 거치며, 유효하지 않거나 만료된 토큰은 `401 Unauthorized`를 반환
- **비정상 행위 차단**: 인가(Authorization) 절차를 통해 본인이 아닌 타인의 정보를 수정하거나, 경매 상태를 임의로 변경하는 행위를 원천 차단

---

## 2. 회원 관리 및 로그인 (Member Management)

### 2.1 회원가입 로직
- **필수 입력값**: 로그인 ID, 비밀번호, 닉네임, 이메일
- **유효성 검사**:
    - 아이디 및 닉네임 중복 체크
    - 이메일 형식 검증
    - 비밀번호 복잡성 규칙 적용 (Zod 등의 스키마 규격 준수)

### 2.2 로그인 프로세스
- **검증**: 입력된 비밀번호와 DB의 암호화된 비밀번호 일치 확인
- **결과**: 인증 성공 시 JWT 발급 및 사용자 프로필 정보(ID, 닉네임, 역할 등) 반환

---

## 3. 마이페이지 및 활동 내역 (MyPage Service)

### 3.1 마이페이지 요약 대시보드
- **통합 카운트 조회**: 유저가 현재 참여 중인 입찰 수, 최종 낙찰받은 물품 수, 본인이 등록한 경매 수, 관심 목록(찜) 등록 수를 한 번에 반환
- **API**: `GET /api/v1/users/me/summary`

### 3.2 개인정보 관리
- **비밀번호 변경**:
    1. 현재 사용 중인 비밀번호 일치 확인
    2. 새로 변경할 비밀번호와 확인용 비밀번호의 일치 여부 검증
    3. 실패 시 전용 예외 처리(비밀번호 불일치 에러) 수행

### 3.3 활동 내역 페이지네이션
- **등록 경매 조회**: 판매자로서 등록한 모든 경매 리스트를 상태별(판매 중, 판매 완료 등)로 필터링하여 페이징 조회
- **입찰/낙찰 조회**: 참여한 경매 중 현재 입찰 중인 물품과 이미 낙찰받은 물품을 구분하여 조회
- **관심 목록**: 찜하기(Like)를 누른 경매 상품 리스트 조회

## 4. 예외 처리 가이드 (Exception Handling)
- **UNAUTHORIZED (401)**: 토큰 만료 또는 인증 실패 시
- **ACCESS_DENIED (403)**: 타인의 정보를 수정하려 하거나 권한 없는 기능을 호출할 때
- **DUPLICATE_RESOURCE (409)**: 이미 존재하는 아이디나 닉네임으로 가입 시도 시
