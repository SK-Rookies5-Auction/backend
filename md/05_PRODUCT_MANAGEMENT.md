# 📦 05. 경매 물품 전시 및 관리 상세 명세 (Product Management & Notification)

본 문서는 백엔드 서버의 경매 물품 등록, 이미지 관리(S3), 카테고리 필터링 API, 상세 정보 조회 및 실시간 알림 스케줄러 로직을 정의합니다.

---

## 1. 경매 물품 등록 및 멀티미디어 관리 (Registration & S3)

### 1.1 경매 등록 프로세스
- **입력 항목**: 경매 제목, 상품 상세 설명, 카테고리 선택(ENUM), 경매 시작가, 경매 시작 예약 시간, 경매 마감 시간.
- **이미지 업로드 로직**:
    - **AWS S3 연동**: 클라이언트로부터 이미지를 전달받아 서버 로컬에 남기지 않고 S3 버킷에 직접 업로드 처리합니다.
    - **Picture 테이블 관리**: 업로드 성공 후 반환된 S3 전체 URL과 관리용 S3 Object Key를 DB에 기록합니다.
    - **대표 이미지**: 업로드된 사진 중 하나를 대표 사진(`is_main=true`)으로 식별하여 저장합니다.
    - **순서 관리**: 클라이언트가 요청한 사진의 출력 순서(`sort_order`)를 함께 저장합니다.

---

## 2. 경매 리스트 및 탐색 API (Product Listing API)

### 2.1 메인 및 카테고리 화면 데이터 제공
- **페이징 적용**: 대량의 물품 조회를 위해 `Pageable`을 활용하여 페이지네이션(또는 무한 스크롤용 커서 기반 페이징) 응답을 구현합니다.
- **필터링 및 정렬 (Querydsl 등 활용 권장)**:
    - **카테고리**: WATCH, ART, ELECTRONIC 등 요청된 카테고리별 필터링을 지원합니다.
    - **정렬**: 최신순, 마감 임박순, 인기순(찜 순), 조회수순 등으로 데이터를 정렬하여 반환합니다.
- **응답 데이터**: 각 상품 목록 응답 시 현재 최고가(`current_price`), 경매 상태(`status`), 마감 시간(`end_time`)을 반드시 포함해야 합니다.

---

## 3. 경매 상세 정보 및 커뮤니케이션 API (Detail View & Q&A)

### 3.1 상세 정보 응답 구성
- **물품 및 셀러 정보**: 상품의 전체 사진 리스트, 상세 설명, 판매자의 닉네임 정보를 조인하여 반환합니다.
- **통계 데이터 업데이트**: 상세 조회 API 호출 시 해당 경매의 조회수(`view_count`)를 증가시키는 로직을 포함합니다.

### 3.2 Q&A (문의 게시판) API
- **댓글 시스템**: `comments` 테이블을 활용하여 문의 등록 및 조회 API를 구현합니다.
- **작성 권한**: `SecurityContext`를 통해 인증된 사용자만 문의를 등록할 수 있도록 접근을 제어합니다.

## 4. 실시간 알림 서비스 로직 (Notification Scheduler & API)

### 4.1 알림 발생 조건 및 트리거 로직
- **상위 입찰 발생 (OUTBID)**:
    - 트리거: 입찰 API(`POST /bids`) 호출 시, 기존 최고가 입찰자가 존재하면 해당 유저를 대상으로 알림 생성 로직을 비동기로 호출합니다.
    - 내용: "[상위 입찰] '경매명'에 더 높은 가격이 제시되었습니다."
- **마감 임박 알림 (CLOSING_SOON)**:
    - 트리거: 백엔드 **스케줄러(@Scheduled)** 가 주기적으로 실행되어 `(end_time - 현재 서버 시간) <= 1시간`인 경매를 찜한 유저를 찾아 알림을 생성합니다.
    - 내용: "[마감 임박] 찜하신 '경매명'의 마감이 1시간 남았습니다."
- **낙찰 결과 알림 (AUCTION_RESULT)**:
    - 트리거: 마감 스케줄러에 의해 경매 상태가 `FINISHED`로 업데이트되는 트랜잭션 내에서 최종 낙찰자와 판매자에게 알림을 생성합니다.

### 4.2 알림 데이터 구조 및 제공 API
- **엔티티 구성**: `id`, `type`(OUTBID 등), `content`, `target_url`, `is_read`(기본 false), `created_at`.
- **조회 API**: 로그인한 유저의 최신순 알림 목록을 반환하는 `GET /api/v1/notifications` 구현.
- **읽음 처리 API**: 특정 알림의 `is_read` 상태를 true로 변경하는 `PATCH /api/v1/notifications/{id}` 구현.
