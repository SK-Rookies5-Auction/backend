# 💡 03. 핵심 비즈니스 로직 및 보안 정책 상세 (Core Features & Security)

본 문서는 실시간 경매 플랫폼의 비즈니스 무결성과 보안을 보장하기 위한 핵심 로직 및 정책을 정의합니다. AI는 모든 기능 개발 시 본 문서의 정책을 최우선으로 준수해야 합니다.

---

## 1. 백엔드 핵심 비즈니스 로직 (Backend Core Logic)

### 1.1 실시간 입찰 및 동시성 제어
- **데이터 무결성 보장**: 마감 직전 트래픽 폭주 시 데이터 무결성 파손을 방지하기 위해 MariaDB의 **낙관적 락(Optimistic Lock)**을 적용합니다.
- **입찰 순차 처리**: 수많은 유저가 동시에 입찰하더라도 데이터 꼬임이나 초과 입찰 없이 안전하게 순차 처리되는 락 로직을 구현합니다.

### 1.2 정교한 입찰 검증 (Validation)
- **금액 검증**: 현재 최고가(current_price)보다 높은 금액만 승인하며, 음수 입력 및 0원 입찰을 철저히 방지합니다.
- **본인 입찰 제한**: 판매자(seller_id)는 본인이 등록한 경매 물품에 직접 입찰할 수 없도록 원천 차단합니다.
- **상태 검증**: 경매 상태가 'LIVE'인 경우에만 입찰을 허용합니다.

### 1.3 경매 상태 및 자동화 관리
- **자동 마감 스케줄러**: 경매 마감 시간(end_time)에 도달하면 백엔드 스케줄러가 자동으로 상태를 **CLOSED(종료)**로 변경하고 최종 낙찰자를 확정합니다.
- **거래 상태 자동 전이**:
    - 낙찰자 결제 완료 시: `PAID` 상태로 업데이트
    - 판매자 배송 시작 시: `SHIPPING` 상태로 업데이트
    - 낙찰자 수령 확인 시: `COMPLETED` 상태로 최종 완료

## 2. 보안 및 네트워크 정책 (Security & Network)

### 2.1 인증 및 인가 (Auth)
- **JWT 보안**: Spring Security와 JWT 기반으로 로그인 및 토큰 발급을 처리합니다.
- **비밀번호 암호화**: BCrypt를 이용한 단방향 암호화를 반드시 적용합니다.
- **권한 제어**: API 호출 시 인터셉터/필터를 통해 토큰을 검증하고, 비인가 접근(예: 타인의 입찰 임의 취소, 권한 없는 상태 변경)을 차단합니다.

### 2.2 인프라 및 네트워크 보안
- **VPC 격리**: 퍼블릭 서브넷(ALB)과 프라이빗 서브넷(EKS, RDS)을 완벽히 격리하여 DB 직접 접근을 차단합니다.
- **WAF 연동**: AWS WAF를 적용하여 SQL Injection, XSS 및 악성 매크로 트래픽을 방어합니다.
- **민감 정보 보호**: DB 비밀번호, JWT 시크릿 키 등은 **AWS SSM Parameter Store**를 통해 관리하며 소스 코드에 절대 하드코딩하지 않습니다.

## 3. 배포 및 성능 가용성 (DevOps)

### 3.1 무중단 배포 (CI/CD)
- **GitOps 파이프라인**: GitHub Actions와 ArgoCD를 연동하여 EKS 클러스터에 컨테이너를 자동 배포합니다.
- **무중단 업데이트**: Rolling Update 전략을 통해 배포 중에도 경매 서비스 다운타임이 발생하지 않도록 구성합니다.

### 3.2 성능 모니터링 및 자원 제어
- **자원 제한**: EKS Resource Quota를 설정하여 입찰 서버에 트래픽이 폭주하더라도 다른 서비스(상품 조회 등)가 다운되지 않도록 격리합니다.
